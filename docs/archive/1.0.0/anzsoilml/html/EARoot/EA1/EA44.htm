A SpatialEntity is part of a soil description. <br /></p><p>It provides a link between a notional feature (description package) and one spatial representation of it, or part of it. (Sites, Transects, Exposures, Surface Traces and Intercepts, etc)<br /></p><p>    * the specific bounded occurrence, such as a map polygon<br /></p><p>    * the SpatialEntity carries a geometry or shape<br /></p><p>          - the association with a Soil or Landscape Feature provides the specification of all the other descriptors<br /></p><p>          - the association with a Sampling Feature provides the context and dimensionality <br /></p><p><br /></p><p>A SpatialEntity is always associated with some sampling feature - e.g. a Soil Sample, a Landscape, a Soil Layer etc.  <br /></p>

Notes: SpatialEntity observationMethod is a metadata snippet indicating how the spatial extent of the mapped feature was determined. ObservationMethod is a convenience property that provides a quick and dirty approach to observation metadata when data are reported using a feature view (as opposed to observation view). This property corresponds (loosely) to ISO19115 Lineage.  (eg: digitised, Global Positioning System, published map, field observation,  aerial photography, field survey
